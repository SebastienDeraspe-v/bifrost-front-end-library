# Que faire quand une demande arrive?

Annexes à la demande:
- Figma: Mockup
- [optionnel] Zeroheight: Lien vers les pages qui ont changé dans Zeroheight.
- [optionnel] Storybook: Lien vers les composants qui ont changé/a integrer.



Si la demande concerne une composante Front-End existant:

Guideline depuis le depot DRUPAL
- Modifier la config Drupal.
- Faire l'implementation et l'intégration front-end directement dans Drupal (et non plus plus via un template dans patternlab/storybook)
- Inclure les components n/cessaire
- QA local - Push 
- PR dans drupal (ajouter un dev du bifrost comme reviers)
- Creation du PR preview (URL a partager au designer/stake holder pour valider)
- (step automatise maybe) Réaliser un snapshot (via Backstop?) pour afficher les différences visuelles avec la prod.


Si la demande concerne/possede une NOUVELLE composante Front-End:

Guideline depuis le depot bitbucket DRUPAL
(same as before)

Guideline depuis le depot bitbucket BIFROST
- Cloner depuis la branche XXXXX
- Creation de la nouvelle composante avec la commande CLI du bifrost `npm generate component XXXXX`
- Integrer le code HTML dans le ficheir .twig correspondant du nouveau dossier genere par la commande precedente
- Valider l,accesssibilite et la fonctionnalit/ de la composante
